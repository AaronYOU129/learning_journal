## 📘 What I Learned Today  

### Prompt Engineering — Expanding & Chat-based Interaction  

---

### (1) Expanding / Rewriting  

**Use Case:**  
LLMs can expand or refine text, such as polishing emails or improving clarity and tone.  

**Example:**  
- Rewrite emails in a more professional tone  
- Expand short notes into full explanations  

---

### (2) Temperature  

**Definition:**  
Temperature controls the randomness of the model’s output.  

- **High temperature** → more diverse and creative responses  
- **Low temperature** → more deterministic and consistent responses  

**Insight:**  
Temperature balances creativity and reliability.  

---

### (3) Roles in Chat-based Models  

- **System**: defines behavior, tone, and constraints  
- **User**: provides instructions  
- **Assistant**: generates responses  

**Insight:**  
The system role is key to controlling model behavior across the conversation.  

---

### (4) Task-specific Chatbot Design  

LLMs can be used to build customized assistants for specific tasks:  
- Email assistant  
- Customer support  
- Research assistant  

**Insight:**  
Prompt + role design = task-oriented AI system  

---

### (5) Context Engineering  

**Definition:**  
Context engineering refers to structuring and managing the input context  
so that the model produces better and more relevant outputs.  

**Key Techniques:**  
- Using delimiters (e.g., ``` ``` ) to separate data  
- Providing examples (few-shot prompting)  
- Supplying relevant background information  
- Structuring inputs clearly  

**Insight:**  
While prompt engineering focuses on instructions,  
context engineering focuses on **what information the model sees and how it is organized**.  

---

### ✅ Key Takeaway  

- Prompt engineering = control instructions  
- Context engineering = control information  

👉 Together, they control **model behavior and output quality**
